## 2024-03-21 - [DOM Sanitization Bypass]
**Vulnerability:** XSS through unhandled dangerous DOM elements (`<script>`, `<style>`, `<iframe>`, `<math>`, `<svg>`) inside `SocialComments.astro`. The custom DOM sanitizer used an allowlist to unwrap dangerous tags by iterating their children, but didn't completely discard them and then still used `innerHTML`. This pattern leaves open the potential to reconstruct dangerous nodes or execute script inside SVG/Math contexts.
**Learning:** Whitelists alone are insufficient when sanitizing generic HTML from external sources (e.g. Mastodon API) if the implementation merely unwraps unallowed tags. Aggressively discarding dangerous parent nodes completely prevents nested bypasses.
**Prevention:** Always pair an `ALLOWED_TAGS` whitelist with a `DISCARD_TAGS` list (`SCRIPT`, `STYLE`, `IFRAME`, `SVG`, `MATH`) to halt recursion early. Universally replace `innerHTML` assignments with native DOM methods (`document.createElement`, `textContent`, `appendChild`) to guarantee inputs are parsed as data, not executable code.
