## 2025-03-22 - Whitelist DOM Sanitization Defense in Depth
**Vulnerability:** Whitelist-based DOM sanitizers that only use an ALLOWED_TAGS array to determine whether to copy an element or just unwrap its children (e.g. keeping text content) can still be bypassed or lead to unexpected DOM structure if hostile elements (like <script>, <style>, <iframe>, <math>) are left intact (unwrapped, or otherwise rendering unexpectedly). Additionally, using `innerHTML` to populate loading and error messages in the DOM enables potential XSS vectors if those strings ever incorporate untrusted input.
**Learning:** Defense in depth dictates using both an ALLOW_TAGS and a DISCARD_TAGS list. Known dangerous elements must be aggressively dropped instead of unwrapped or evaluated later. Additionally, avoiding `innerHTML` universally across client-side scripts ensures that changes to the component structure or future extensions will not suddenly create sinks for Cross-Site Scripting.
**Prevention:** Use `document.createElement`, `textContent`, and `replaceChildren` / `appendChild` rather than setting `innerHTML`. Explicitly filter dangerous nodes in sanitizers by returning `null` outright.
